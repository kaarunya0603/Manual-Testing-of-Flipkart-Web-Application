## 1. Introduction
This test plan describes the scope, approach, and objectives of manual testing performed on a live e-commerce web application (Flipkart) for learning purposes.

## 2. Objective
The objective of this testing is to ensure that key e-commerce functionalities work as expected and provide a smooth user experience.

## 3. Scope of Testing
### In-Scope
- Login functionality
- Product search
- Search filters and sorting
- Product Details Page (PDP)
- Add to Cart and Cart page

### Out-of-Scope
- Payment processing
- Order placement
- Refunds and cancellations

## 4. Test Approach
- Manual functional testing
- UI/UX validation
- Exploratory testing

## 5. Test Environment
- Application: Flipkart (Web)
- Browser: Google Chrome
- Platform: Windows

## 6. Test Deliverables
- Test cases document
- Bug reports
- Test plan document

## 7. Entry Criteria
- Application is accessible
- Test environment is stable

## 8. Exit Criteria
- All planned test cases are executed
- Defects are documented

## 9. Roles and Responsibilities
- Test Planning: Self
- Test Case Design: Self
- Test Execution: Self
- Defect Reporting: Self

## 10. Risks and Assumptions
- Application behavior may change as it is a live system
- Testing is limited to publicly accessible features

## 11. Approval
This test plan is created for educational and portfolio purposes only.
